# Repository structure

**Competition reference:** [151B_SP26_Competition](https://github.com/brooksniu/151B_SP26_Competition) ŌĆö starter template, dataset narrative, loading patterns. Task definition and schema are summarized in `PROJECT_BRIEF.md` and `DATASETS.md`.

**Environment:** `environment.yml` (full GPU stack; Windows-friendly), `environment-vllm.yml` (adds vLLM on Linux/WSL), `requirements.txt` (venv), `scripts/register_jupyter_kernel.py`, and `ENVIRONMENT_SETUP.md`.

**Inference notebook (implemented):** `notebooks/02_inference.ipynb` ŌĆö see `STATUS.md` for the full change list. High level: required `Qwen/Qwen3-4B-Thinking-2507`, repo-relative paths, public/private `DATA_MODE`, adaptive vLLM generation, public scoring, private JSONL save, and quoted CSV export.

## Layout

- `configs/` ŌĆö run configuration (`default.yaml`; not yet wired into notebook)
- `context/` ŌĆö documentation and project memory (this file, rules, briefs)
- `data/raw/` ŌĆö competition JSONL (gitignored; use `.gitkeep` for empty dir)
- `data/external/` ŌĆö sample submission and other external references
- `notebooks/` ŌĆö EDA and inference
- `src/` ŌĆö reusable library code (scaffold / TBD)
- `scripts/` ŌĆö CLI utilities (e.g. submission CSV, `register_jupyter_kernel.py`)
- `artifacts/logs/runs/` ŌĆö per-run JSONL and metadata (gitignored contents)
- `artifacts/submissions/` ŌĆö CSVs for leaderboard upload
